## Architecture

Client-centric SPA with declarative routing. State managed via Zustand stores, data persisted through Supabase client SDK. Component-based UI with Tailwind CSS for styling.

```
src/
├── main.tsx                # Vite entry point
├── App.tsx                 # Root component
├── AppRoutes.tsx           # Client-side routing
├── store/                  # Zustand state stores
│   ├── productsStore.ts
│   └── authStore.ts
├── lib/                    # Helpers and clients
│   ├── supabase.ts         # Supabase client wrapper
│   └── utils.ts            # Utility functions
├── components/             # Reusable UI components
├── pages/                  # Route-level components
├── services/               # API service layer (placeholder)
└── types/                  # TypeScript type definitions
```

- **User** – authenticated users via Supabase Auth
- **Stock Transaction** – record of stock changes

### Business Rules
